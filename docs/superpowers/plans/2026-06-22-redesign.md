# UI Redesign Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers-extended-cc:subagent-driven-development (recommended) or superpowers-extended-cc:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the ChatGPT-style glassmorphism dark UI with a clean, light, utility-focused design using indigo as the single accent colour.

**Architecture:** Pure static site — `style.css` (all visual tokens and component styles), `index.html` (structure and new HTML for progressive disclosure), `app.js` (interaction logic for collapsible sections and format strip). No build step, no dependencies beyond the vendored PDF libs.

**Tech Stack:** Vanilla HTML/CSS/JS. Open `index.html` directly in any browser to verify.

**User decisions (already made):**
- Direction: Clean Utility — white background, no gradients, no glassmorphism
- Accent: Indigo `#4f46e5`
- Corner radius: Moderate — 6–8 px for inputs/buttons, 12 px for app shell
- Aufteilung custom fields (Zeilen/Spalten/Seiten) hidden until "Eigene…" is clicked
- Ausgabeformat as button strip; default = Kachelgröße; W/H fields hidden until "Eigenes…" clicked
- Rand + Überlappung hidden behind "Erweitert ▾" toggle
- Footer: `v0.1.0` links to `https://github.com/si0nDE/cutpdf.app`
- Header: no version number top-right
